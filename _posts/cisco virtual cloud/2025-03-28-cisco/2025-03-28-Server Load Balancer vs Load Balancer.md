# **서버 로드 밸런서(Server Load Balancer, SLB)와 로드 밸런서(Load Balancer)의 차이점**



## **1. 개념 정의**

------

**로드 밸런서(Load Balancer, LB)**:

→ 네트워크 트래픽을 여러 대상(서버, 애플리케이션, 데이터센터 등)으로 분산하여 부하를 줄이고 성능을 최적화하는 장치 또는 서비스)

**서버 로드 밸런서(Server Load Balancer, SLB)**:

→ 로드 밸런서의 한 유형으로, **특히 서버 간의 부하 분산을 담당하는 로드 밸런서**



------



**2. 주요 차이점 비교**

| **구분**      | **서버 로드 밸런서 (SLB)**                                   | **일반 로드 밸런서 (LB)**                                    |
| ------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **대상**      | **서버 간 부하 분산**                                        | 네트워크 장비, 서버, 애플리케이션 등 여러 대상에 부하 분산   |
| **사용 목적** | 서버 리소스 최적화 및 부하 분산                              | 네트워크 트래픽 최적화 및 다양한 리소스 부하 분산            |
| **계층**      | 주로 L4(전송 계층) 또는 L7(애플리케이션 계층)에서 작동       | L3(네트워크 계층), L4, L7에서 작동 가능                      |
| **예제**      | - 웹 서버(Apache, Nginx) 부하 분산 - 데이터베이스 서버(MySQL, PostgreSQL) 부하 분산 | - 네트워크 트래픽 부하 분산 - 데이터센터 간 부하 분산 (GSLB, Global Server Load Balancer) |
| **기능**      | - 서버 간 트래픽 분산 - 서버 장애 감지 및 대체 - 특정 요청을 특정 서버로 라우팅 | - 방화벽 기능 포함 가능 - 지역 기반 라우팅(Global LB) 가능 - 네트워크 패킷 최적화 가능 |
| **예시 제품** | - AWS ALB(Application Load Balancer), ELB - F5 BIG-IP LTM - Nginx Load Balancer | - DNS 기반 GSLB (Global Server Load Balancer) - 네트워크 로드 밸런서 (Cisco, A10) |





------



**3. 쉽게 이해하는 차이점**

​	•	**SLB(서버 로드 밸런서)**: 서버들 사이에서 부하를 분산하는 로드 밸런서

→ **서버 간 부하를 관리하는 역할**

​	•	**일반 로드 밸런서(LB)**: 서버뿐만 아니라 네트워크 트래픽, 애플리케이션 등 다양한 대상에 부하를 분산

→ **네트워크 전체의 트래픽을 최적화하는 역할**



------



**4. 결론**

​	•	**SLB(Server Load Balancer)**는 **서버 간 부하를 분산**하는 데 특화된 로드 밸런서

​	•	**일반 로드 밸런서(LB)**는 네트워크, 애플리케이션, 데이터센터 등 다양한 트래픽을 분산

​	•	**SLB는 로드 밸런서의 한 종류**이며, 모든 로드 밸런서가 SLB인 것은 아님

​	•	서버 부하 관리가 목적이라면 **SLB**

​	•	네트워크 전반의 트래픽 최적화가 목적이라면 **일반 로드 밸런서**
