Elastic Load Balancer (ELB) - AWS에서 제공하는 로드 밸런서 서비스로, 트래픽을 여러 대상에 분산하여 특정 서버의 과부하를 방지하고 애플리케이션의 가용성과 확장성을 향상시키는 기능을 제공합니다. 

ELB는 EC2 인스턴스, 컨테이너, IP 주소 등 다양한 대상에 걸쳐 트래픽을 분산할 수 있으며, 등록된 대상의 상태를 모니터링하여 건강한 대상에만 트래픽을 라우팅합니다



#### **로드 밸런싱의 목적**

- **트래픽 분산**: 다운스트림 인스턴스(EC2 등)에 부하를 균등하게 분배.
- **고가용성**: 장애 발생 시 자동으로 정상 인스턴스로 트래픽 전환.
- **단일 접점**: 사용자는 하나의 DNS 엔드포인트로 접속 (백엔드 구조 숨김).
- **SSL 종료**: HTTPS 트래픽 암호화/복호화 처리.
- **상태 확인**: 인스턴스 건강 상태 모니터링 (Health Checks).

------

#### **Elastic Load Balancer (ELB) 종류 비교**

| **유형**              | **프로토콜**           | **사용 사례**                              | **특징**                             |
| :-------------------- | :--------------------- | :----------------------------------------- | :----------------------------------- |
| **CLB (Classic)**     | HTTP, HTTPS, TCP, SSL  | 레거시 애플리케이션                        | AWS 비권장 (V1, 2009년 출시)         |
| **ALB (Application)** | HTTP, HTTPS, WebSocket | 웹 애플리케이션 (마이크로서비스)           | 경로/호스트 기반 라우팅, Lambda 통합 |
| **NLB (Network)**     | TCP, TLS, UDP          | 초고속/저지연 트래픽 (게임, 실시간 스트림) | 고정 IP 지원, 초당 100만 요청 처리   |
| **GWLB (Gateway)**    | IP 계층 (Layer 3)      | 방화벽/보안 장비 통합 (3rd-party)          | GENEVE 프로토콜, 트래픽 복제 가능    |

### **차이점 한눈에 보기**

| **구분**      | **ALB (Application)**                   | **NLB (Network)**                         | **CLB (Classic)**           |
| :------------ | :-------------------------------------- | :---------------------------------------- | :-------------------------- |
| **계층**      | **7계층 (HTTP/HTTPS)**                  | **4계층 (TCP/UDP)**                       | **4계층 & 7계층**           |
| **주요 기능** | - 경로/호스트 기반 라우팅 - Lambda 통합 | - 초고속 처리 (초당 100만 요청) - 고정 IP | - 레거시 지원 - 기본 라우팅 |
| **성능**      | 중간 (~10만 RPS)                        | 최고 (100만 RPS+)                         | 낮음 (~5만 RPS)             |
| **SSL 처리**  | O (자동 인증서 관리)                    | X (EC2에서 처리)                          | O                           |
| **가격**      | 요청 수 + 시간당                        | 시간당 + 처리 데이터량                    | 시간당                      |

## **언제 어떤 걸 써야 할까?**

**ALB (Application Load Balancer)**

- **필요할 때**:
  - **웹 애플리케이션** (HTTP/HTTPS).
  - **마이크로서비스** (경로별 라우팅 예: `/api`, `/img`).
  - **서버리스** (Lambda와 통합).

#### **NLB (Network Load Balancer)**

- **필요할 때**:
  - **초저지연**이 필요한 앱 (게임, 실시간 스트리밍).
  - **TCP/UDP** 기반 서비스 (예: 데이터베이스 복제).
  - **고정 IP** 필요 (예: 화이트리스트 등록).

#### **CLB (Classic Load Balancer)**

- **필요할 때**:
  - **레거시 애플리케이션** 유지보수.
  - **EC2-Classic** 환경 사용 시 (새 계정에선 지원 X).
